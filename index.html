Django Setup Guide

Prerequisites

Before we begin, make sure you have the following installed on your system:

  1. Python - You can download it from python.org
    • Make sure to check "Add Python to PATH" during installation
    • Verify installation by running python --version in your terminal
  2. Visual Studio Code - You can download it from code.visualstudio.com
    • This will be our code editor for the Django project
    • Consider installing the Python extension for better Python support

Steps

Step 1: Create an Empty Project Folder

First, we'll create a clean empty folder where we'll be working on our Django project. This will be our project's root directory.

Windows Command Prompt:

mkdir my_django_project
cd my_django_project

Alternative: You can also create the folder using File Explorer:

  1. Navigate to your desired location
  2. Right-click and select "New" > "Folder"
  3. Name it my_django_project (or any name you prefer)
  4. Open the folder

Step 2: Open the Folder in VS Code

After creating your project folder, we'll open it in Visual Studio Code to start working on our Django project.

Method 1: Using Command Line

code .

Note: Make sure you're in your project folder before running this command

Method 2: Using VS Code Interface

  1. Open Visual Studio Code
  2. Click on "File" > "Open Folder..."
  3. Navigate to your my_django_project folder
  4. Click "Select Folder"

Method 3: Using Right-Click (if available)

  1. Right-click on your project folder in File Explorer
  2. Select "Open with Code" (if this option is available)

Step 3: Open Terminal in VS Code

Before we create a virtual environment, we need to open the integrated terminal in VS Code. This terminal will allow us to run Python and Django commands directly within our development environment.

Method 1: Using Keyboard Shortcut

  • Press Ctrl + ` (backtick) on Windows
  • This is the fastest way to open/close the terminal

Method 2: Using Menu Bar

  1. Click on "View" in the top menu bar
  2. Select "Terminal" from the dropdown menu

Method 3: Using Toggle Panel

  1. Look for the toggle panel icon in the top right area of VS Code (it looks like this: ⬜)
  2. Click on the toggle panel icon to open/close the bottom panel
  3. Once the panel is open, click on the "Terminal" tab
  4. Keyboard shortcut: You can also press Ctrl + J to toggle the panel

VS Code Terminal Guide

When you open the terminal in VS Code, it generally defaults to PowerShell on Windows. You can identify the current shell by looking near the + icon in the terminal area. There is a dropdown button next to the + icon that allows you to switch between different terminal types.

Switching to Command Prompt (cmd)

  1. Click on the dropdown button near the + icon in the terminal.
  2. Select Command Prompt (cmd) from the list of available shells.

Why Use Command Prompt (cmd) for Django?

  • Consistency: Most Django tutorials and documentation use cmd for commands.
  • Simplicity: cmd has fewer compatibility issues compared to PowerShell when running Django commands.
  • Avoiding Errors: Some commands may behave differently in PowerShell, leading to confusion for beginners.

While PowerShell works for Django, using cmd ensures a smoother experience, especially for beginners.

Setting Command Prompt as Default Terminal

To make your workflow smoother, you can set Command Prompt (cmd) as the default terminal in VS Code. This ensures that every time you open a new terminal, it defaults to Command Prompt instead of PowerShell or other shells.

How to Set Command Prompt as Default:

  1. Click on the dropdown button near the + icon in the terminal area.
  2. Select "Select Default Profile" from the dropdown menu.
  3. Choose Command Prompt from the list of available profiles.

Benefits of Setting Command Prompt as Default:

  • Consistency: Always starts with the terminal type you prefer for Django development.
  • Time-Saving: No need to manually switch to Command Prompt every time you open a new terminal.
  • Avoids Errors: Reduces the chance of accidentally running commands in the wrong shell (e.g., PowerShell or Bash).

Once set, every new terminal you open will automatically use Command Prompt, making your Django development process more efficient.

Adding Multiple Terminals

You can add multiple terminal instances by clicking the + icon. However, make sure to check which terminal you are using (e.g., PowerShell, Bash, Command Prompt, JavaScript Debug, etc.).

Important Note

While working in Django, always ensure you are using Command Prompt (cmd) to avoid unexpected issues.

Verify Terminal is Working:

Once the terminal is open, you should see a command prompt in your project directory. You can verify by running:

dir

This should show an empty directory (since we just created the folder).

Step 4: Create and Activate a Virtual Environment

A virtual environment is an isolated environment for Python projects. It ensures that dependencies for one project do not interfere with others. This is especially important when working with Django, as it allows you to manage project-specific packages.

Why Create a Virtual Environment?

  • Dependency Management: Keeps project dependencies isolated.
  • Avoid Conflicts: Prevents version conflicts between different projects.
  • Clean Environment: Ensures a clean Python environment for your project.

Creating a Virtual Environment

We'll use the Command Prompt in the VS Code terminal to create the virtual environment.

  1. Open the terminal in VS Code (ensure you're using Command Prompt).
  2. Run the following command to create a virtual environment: 
    python -m venv environmentname

    Replace environmentname with the name you want for your virtual environment (e.g., venv).

Activating the Virtual Environment

To activate the virtual environment, use the following single-line command:

.\environmentname\Scripts\activate

This ensures you don't need to switch directories and avoids confusion.

Example: Terminal Before and After Activating Virtual Environment

Before Activation:

C:\Users\YourUsername\my_django_project>

After Activation:

(environmentname) C:\Users\YourUsername\my_django_project>

Notice the name of the virtual environment (environmentname) appears inside parentheses ( ) before the directory path. This indicates that the virtual environment is active.

Important Notes:

  1. Reactivating in New Terminals: If you open a new terminal, you need to activate the virtual environment again using the same command.
  2. Selecting Python Interpreter: To avoid reactivating manually:
    • Press Ctrl + Shift + P in VS Code.
    • Type "Python: Select Interpreter" and press Enter.
    • Look for the interpreter with the name of your virtual environment folder and select it.
    • Check if the name of your virtual environment appears inside parentheses ( ) before the directory path in the terminal.

Troubleshooting:

  • Caution Symbol in Terminal: If you see a caution symbol (e.g., an exclamation mark) near the terminal, hover over it. If it suggests "Relaunch Terminal," click it. Ensure no process is running in the terminal before relaunching.
  • Improper Setup Issues: If you face issues, ensure the virtual environment was created and activated correctly. Relaunching the terminal or reselecting the Python interpreter can resolve most problems.

Step 5: Download and Install Django

Now that we have our virtual environment set up, the next step is to download and install Django. Before proceeding, ensure that:

  • You are in the main parent directory where the virtual environment folder was created.
  • The virtual environment is activated.

Example Folder Structure

Here’s an example of how your project folder might look at this stage:

my_django_project
│
├── environmentname/   # Virtual environment folder
├── ... (other files or folders may appear later)

Steps to Download Django

  1. Navigate to the Main Parent Directory:
    • If you are not already in the main parent directory, use the following command to navigate there: 
      cd path\to\my_django_project

      Replace path\to\my_django_project with the actual path to your project folder.

  2. Ensure Virtual Environment is Activated:
    • If the virtual environment is not activated, activate it using: 
      .\environmentname\Scripts\activate

      Check that the virtual environment name appears in parentheses ( ) before the directory path in the terminal.

  3. Install Django:
    • Ensure you are in the main parent directory where the virtual environment folder was created.
    • Run the following command to install Django: 
      pip install django

Example: Installing Django in the Main Parent Directory

Assume your project folder is named my_django_project and your virtual environment is named venv. Here’s how the terminal should look:

Before Installing Django:

(venv) C:\Users\YourUsername\my_django_project>

After Installing Django:

(venv) C:\Users\YourUsername\my_django_project> pip install django
Collecting django
  Downloading Django-4.x.x-py3-none-any.whl (8.0 MB)
Installing collected packages: django
Successfully installed django-4.x.x

Verifying Django Installation:

(venv) C:\Users\YourUsername\my_django_project> python -m django --version
4.x.x

This confirms that Django has been successfully installed in your virtual environment.

Common Mistakes When Installing Django

Here are some common mistakes users might make when installing Django, along with examples to help you avoid them. Assume your workspace directory is D:/Workspace/DjangoWorkspace1 and your virtual environment folder is named DjVenv.

  1. Installing Without Activating the Virtual Environment
    • Mistake: Forgetting to activate the virtual environment before running the pip install django command.
    • Example: 
      D:\Workspace\DjangoWorkspace1> pip install django
      • This installs Django globally on your system instead of in the virtual environment, which can lead to conflicts with other projects.
  2. Installing Inside the Virtual Environment Folder
    • Mistake: Navigating into the virtual environment folder (e.g., DjVenv) or its Scripts folder and running the install command there.
    • Example: 
      D:\Workspace\DjangoWorkspace1\DjVenv> pip install django
      or 
      D:\Workspace\DjangoWorkspace1\DjVenv\Scripts> pip install django
      • This can corrupt the virtual environment and cause issues with package management.
  3. Installing Outside the Parent Folder
    • Mistake: Running the install command in a directory outside the project folder, such as the parent folder of the workspace.
    • Example: 
      D:\Workspace> pip install django
      • This installs Django in the wrong location, making it unavailable for your specific project.

How to Avoid These Mistakes

  • Always ensure you are in the main parent directory (e.g., D:/Workspace/DjangoWorkspace1) where the virtual environment folder is created.
  • Activate the virtual environment before installing Django: 
    .\DjVenv\Scripts\activate
  • Verify the terminal prompt shows the virtual environment name in parentheses (DjVenv) before the directory path.
  • Run the install command only in the main parent directory: 
    pip install django

Step 6: Create a Django Project

Now that Django is installed, let’s create a Django project. You have two options:

Option 1: Using the . (Dot) Notation

  • Command: 
    django-admin startproject projectname .
  • What It Does: Creates the project files directly in the current directory without creating an additional folder for the project.
  • Recommended: Yes, because it avoids unnecessary nested folders and simplifies navigation.

Option 2: Without Using the . (Dot) Notation

  • Command: 
    django-admin startproject projectname
  • What It Does: Creates a new folder named projectname inside the current directory and places all project files inside it.
  • Recommended: No, as it can lead to confusion about whether you are working inside or outside the project folder.

Example Folder Structures

Using the . (Dot) Notation:

my_django_project
│
├── DjVenv/          # Virtual environment folder
├── manage.py        # Django management script
├── projectname/     # Django project folder
│   ├── __init__.py
│   ├── asgi.py
│   ├── settings.py
│   ├── urls.py
│   └── wsgi.py

Without Using the . (Dot) Notation:

my_django_project
│
├── DjVenv/          # Virtual environment folder
├── projectname/     # New folder created for the project
│   ├── manage.py    # Django management script
│   ├── projectname/ # Django project folder
│   │   ├── __init__.py
│   │   ├── asgi.py
│   │   ├── settings.py
│   │   ├── urls.py
│   │   └── wsgi.py

Why Use the . (Dot) Notation?

  • Avoids unnecessary folder nesting.
  • Simplifies navigation and file management.
  • Recommended for clarity and ease of use.

Step 7: Create a Django App

Once your Django project is set up, the next step is to create an app. Just like when creating a project, you have two options for how you structure the app creation.

Option 1: Without Using the . (Dot) Notation

  • Command: 
    python manage.py startapp your_app_name
  • What It Does: Creates a new folder named your_app_name in the current directory and places all app-related files inside it.
  • Recommended: ✅ Yes, because it keeps your app self-contained, easy to manage, and follows Django best practices.

Option 2: Using the . (Dot) Notation

  • Command: 
    python manage.py startapp your_app_name .
  • What It Does: Places app files (e.g., views.py, models.py, etc.) directly in the current directory, without creating a dedicated folder.
  • Recommended: ❌ No, because it scatters the app files into the root of your project, causing poor organization and potential conflicts.

Example Folder Structures

Without Using the . (Dot) Notation:
Suppose your app name is blog: 

my_django_project
│
├── manage.py
├── projectname/
│   └── settings.py
├── blog/             # App folder
│   ├── admin.py
│   ├── apps.py
│   ├── models.py
│   ├── views.py
│   └── ...

Using the . (Dot) Notation:
Suppose your app name is blog: 

my_django_project
│
├── manage.py
├── projectname/
│   └── settings.py
├── admin.py           # App files scattered in root
├── apps.py
├── models.py
├── views.py
└── ...

Why Avoid the . (Dot) Notation for Apps?

  • Breaks the separation of concerns (no isolated app folder).
  • Makes the project root cluttered and harder to manage.
  • Violates Django’s recommended project structure.

Step 8: Configure Django Settings

After creating your app, you need to configure the Django settings to include your app.

Adding Your App to Installed Apps

  1. Open the settings.py file located in your project folder (e.g., projectname/settings.py).
  2. Look for the INSTALLED_APPS list.
  3. Add your app name (e.g., your_app_name) to the list. For example: 
    INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'your_app_name',  # Add your app here
]

Debugging and Allowed Hosts

  1. Ensure DEBUG is set to True during development: 
    DEBUG = True
  2. Add your development server to the ALLOWED_HOSTS list: 
    ALLOWED_HOSTS = ['*']

Warning: Using '*' allows requests from any host. This is fine for development but should never be used in production as it poses a security risk. Always specify allowed hosts in production.

Benefits of Using '*' in Development:

  • If you want to test your Django project on other devices (e.g., laptops, mobile phones) connected to the same Wi-Fi network, using '*' in ALLOWED_HOSTS makes it possible.
  • It allows you to access your project by entering your computer's local IP address in a browser on another device.
  • This is especially useful for testing responsiveness and functionality across different devices during development.

Step 9: Run Migrations and Start the Development Server

After configuring your settings and creating your app, the next step is to apply migrations and start the development server.

Step 9.1: Run Migrations

Migrations are used to apply changes to your database schema. Django provides two commands for this:

  1. Make Migrations:
    • This command generates migration files based on the changes in your models.
    • Run the following command in the terminal: 
      python manage.py makemigrations
    • What It Does: Creates migration files in your app's migrations/ folder.
  2. Apply Migrations:
    • This command applies the migration files to your database.
    • Run the following command: 
      python manage.py migrate
    • What It Does: Updates the database schema based on the migration files.

Step 9.2: Start the Development Server

Once migrations are applied, you can start the development server to test your project.

  1. Run the following command: 
    python manage.py runserver
  2. What It Does: Starts a local development server at http://127.0.0.1:8000/.
  3. Open a web browser and navigate to the URL provided in the terminal (e.g., http://127.0.0.1:8000/).
  4. Check if the Django rocket appears on the page. This indicates that your Django project is set up correctly and the server is running successfully.

Important Notes:

  • Directory: Ensure you are in the root directory of your Django project (the folder containing manage.py).
  • Virtual Environment: Make sure your virtual environment is activated before running these commands. If not, activate it using: 
    .\environmentname\Scripts\activate
  • Troubleshooting:
    • If you encounter errors, check that your app is added to INSTALLED_APPS in settings.py.
    • Ensure the database is properly configured in settings.py (default is SQLite).

Step 10: Create a Basic Landing Page

Now that your Django project is running, let’s create a simple home page or landing page for your project. This involves creating a urls.py file in your app, updating the project’s URL configuration, defining a view, and testing the page.

Step 10.1: Create urls.py in Your App

  1. Navigate to your app folder (e.g., your_app_name).
  2. Create a new file named urls.py.

Step 10.2: Add App URLs to Project URLs

  1. Open the urls.py file in your project folder (e.g., projectname/urls.py).
  2. Import the include function from django.urls: 
    from django.urls import path, include
  3. Add a new path to include your app’s URLs. Replace your_app_name with the name of your app: 
    urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('your_app_name.urls')),  # Include app URLs
]

Step 10.3: Define the View

  1. Open the views.py file in your app folder (e.g., your_app_name/views.py).
  2. Add the following code to define the home view: 
    from django.http import HttpResponse

def home(request):
    return HttpResponse('<h1>Welcome to Your Django Project!</h1>')

Step 10.4: Configure App URLs

  1. Open the urls.py file in your app folder (e.g., your_app_name/urls.py).
  2. Add the following code to define the URL patterns for your app: 
    from django.urls import path
from . import views

urlpatterns = [
    path('', views.home, name='home'),  # Home page URL
]

Step 10.5: Migrate and Run the Server Again

  1. Before remigrating, check if there is already a migration process running:
    • Look at the terminal to see if a migration command is still active.
    • If a migration is running, stop it by pressing Ctrl + C in the terminal.
  2. Follow Step 9 to run the following commands:
    • Apply migrations: 
      python manage.py makemigrations
python manage.py migrate
    • Start the development server: 
      python manage.py runserver
  3. Open a web browser and navigate to http://127.0.0.1:8000/.
  4. You should see the message "Welcome to Your Django Project!" displayed on the page.
